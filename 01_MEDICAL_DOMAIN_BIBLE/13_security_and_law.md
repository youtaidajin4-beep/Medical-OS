# このドキュメントの目的

Medical OSは医療情報を扱うシステムである。

「便利だから作る」ではなく、**「安全だから使える」**ことが最優先となる。

本章では

- 法律
- セキュリティ
- AIの責任範囲
- 開発時の設計原則

を定義する。

---

# Medical OSの最優先事項

優先順位は以下とする。

1. **患者安全**
2. **医療安全**
3. **個人情報保護**
4. **医師の判断支援**
5. **業務効率化**

効率化のために安全性を犠牲にしてはならない。

---

# AIの役割

Medical OSは**診断AI**ではない。**診療情報整理AI**である。

**担当する**

- 情報整理
- 文書生成
- 要約
- 検索
- 過去情報整理

詳細: [`12_medical_dx.md`](./12_medical_dx.md)（付録: 診断 · AIの境界）

---

# AIが絶対にやってはいけないこと

AIは

- 診断を決定しない
- 薬を決定しない
- 検査を決定しない
- 治療方針を決定しない
- 患者へ医療判断を伝えない
- カルテを勝手に確定しない
- **事実を創作しない**

---

# Human in the Loop

Medical OSでは**すべての文書に医師確認を必須**とする。

```
生成
  ↓
確認
  ↓
修正
  ↓
承認
  ↓
電子カルテ保存
```

**AIだけで完結することは禁止。**

---

# ハルシネーション対策

生成AIは存在しない情報を作る可能性がある。

Medical OSでは以下を徹底する。

- 診療音声に存在しない内容を書かない
- 診断を推測しない
- 薬剤を推測しない
- 検査値を推測しない
- 日時を推測しない
- 患者発言を創作しない

不明な場合は **「記載なし」** **「確認が必要」** として出力する。

---

# 個人情報

Medical OSが扱う情報（氏名 · 住所 · 電話番号 · 生年月日 · 診断 · 検査結果 · 画像 · 紹介状 · 処方 · カルテ など）は**要配慮個人情報**である。

**最高レベルの管理**を行う。

---

# データ保存方針

**Version 0.1**

| データ | 方針 |
|--------|------|
| 音声 | 処理 → **自動削除**（設定可能） |
| 文字起こし | 保存 |
| SOAP | 保存 |
| 修正後SOAP | 保存 |
| 診療記録 | 保存 |
| 生成履歴 | 保存 |

Version 0.1 検証: **匿名化 · 模擬データ**（Constitution）

---

# 匿名化

AI学習には**匿名データのみ**利用する。

氏名 · 住所 · 電話番号 · 患者ID · 保険番号 などは**学習対象に含めない**。

---

# 再学習

患者データをAIモデルの再学習へ利用してはならない。

API利用時も**再学習OFF**を前提とする（**要確認**: 契約条項）。

---

# アクセス権限

管理者 · 医師 · 看護師 · 薬剤師 · 医療事務 — それぞれ**閲覧権限を分離**する。

**必要最小限**の情報のみ閲覧可能とする。

---

# ログ管理

以下は必ず記録する。

- ログイン
- 閲覧
- 編集
- 削除
- AI生成
- 承認
- ダウンロード
- コピー

これらは**監査ログ**として保持する。

---

# 録音

診療録音は以下を明確にする。

- 患者への説明
- 患者の同意
- 院内掲示
- 利用目的
- 保存期間
- 削除ルール

**要確認**: 個別同意 vs 院内掲示

---

# 暗号化

| 対象 | 方式 |
|------|------|
| 通信 | TLS |
| 保存 | AES-256 |
| バックアップ | 暗号化 |

を基本とする。

---

# バックアップ

診療情報は

- 自動バックアップ
- 世代管理
- 復元確認

まで行う。

---

# AI品質管理

Version 0.1 では以下を成功条件とする。

| 指標 | 目標 |
|------|------|
| カルテ作成時間 | 5分 → **2分以内** |
| 重大誤認 | **0件** |
| 薬剤誤り | **0件** |
| 数値誤り | **0件** |
| 日付誤り | **0件** |
| 否定表現誤り | **0件** |
| 医師満足度 | **80%以上** |

---

# 医療AIの原則

AIは医師を置き換えない。  
AIは医師の思考を支援する。

**責任は常に医療者にある。**

Medical OSはその責任を支えるためのツールである。

---

# Core Principle

Medical OSが守るべき価値観

- 患者第一
- 医療安全第一
- 情報の正確性
- 透明性
- 説明可能性
- **Human in the Loop**
- **Never Guess**
- **Always Verify**

これらはすべての**実装 · 設計 · UI · AI生成**に優先して適用される。

---

# 補足: 紙カルテ・OCR（将来）

```
紙 → スキャン → OCR → 人間確認 → タイムライン
```

- 手書き誤読リスク高
- 薬剤名 · 投与量 · 日付の誤りは重大
- OCR結果を**正式記録として自動確定しない**
- 重要情報は必ず人が確認

詳細: [`10_medical_documents.md`](./10_medical_documents.md)
