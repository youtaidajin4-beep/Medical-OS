# このドキュメントの目的

Medical OSは**「すべての医療従事者向け」**のプロダクトではない。

最初から全員をターゲットにすると、**誰にも刺さらない**プロダクトになる。

本ドキュメントでは、

- 誰を最初のユーザーにするのか
- なぜその人なのか
- どのような順番で市場を広げるのか

を定義する。

---

# 最初のターゲット

## Version 0.1

**対象**: 内科クリニックの**院長**

**理由**

- 診療する
- カルテを書く
- 紹介状を書く
- 診断書を書く
- 経営もする

**一番困っている人**だからである。

# 先生が毎日感じていること

診療が終わった。でも**仕事は終わらない**。

- カルテ
- 紹介状
- 診断書
- 書類
- 電話
- 確認

まだ何時間もある。**これが日常**である。

---

# 本当の悩み

カルテを書くことではない。

- 患者が増えている
- 書類が増えている
- 紹介状が増えている
- 昔のカルテが探せない
- 診療中に整理できない

**情報量**が限界になっている。

---

# Medical OSが提供する価値

**Before**

```
診療 → カルテ → 紹介状 → 診断書 → 説明 → 残業
```

**After**

```
診療 → Medical OS → 情報整理 → カルテ完成 → 定時帰宅
```

---

# 初期ユーザー像

このような先生を探す。

- AIに興味がある
- 改善意欲が高い
- 毎日診療する
- フィードバックをくれる
- 共同開発に協力してくれる

**売上より協力者を優先**する。

# なぜ病院から始めないのか

病院は

- 導入まで時間がかかる
- 意思決定者が多い
- 電子カルテ変更が難しい
- セキュリティ要件が高い

**PMFには向かない。** まず**意思決定が早いクリニック**から始める。

---

# なぜ地方なのか

Medical OSは**東京から始めない。長崎から始める。**

**理由**

- 先生との距離が近い
- 毎週改善できる
- 現場へ行ける
- 信頼関係が作れる

**地方だからこそ共同開発できる。**

---

# 顧客ではなく共同開発者

最初の先生は**顧客ではない。共同開発者**である。

私たちは製品を売るために会うのではない。  
**一緒に作るために会う。**

---

# PMFの判断

先生が「導入したい」では**弱い**。

本当のPMFは、**「今日はMedical OSを開いてない」と不安になる**状態。

---

# Version 0.1で追わない顧客

以下は**PMF後**。

- 大学病院
- 総合病院
- 行政
- 自治体
- 介護施設
- チェーン病院
- 海外

---

# Core Principle

私たちは**市場を取りに行かない。一人の先生を幸せにする。**

その先生が**次の先生を連れてきてくれる。**

これがMedical OSの成長戦略である。
